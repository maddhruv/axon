# Axon v2 - AI Memory Engine Redesign

## Summary

Full redesign of axon from a note-taking CLI into a true AI memory system. Two-tier architecture (auto-generated briefing + raw archive), BM25 retrieval, system-level memory categories, and a Claude Code skill that auto-captures memories from conversations, consolidates on session end, and injects curated context at conversation start.

The differentiator: markdown-first, git-trackable, no cloud dependency, learns from AI corrections automatically, consolidates via LLM into curated briefings.

## Tier 1: The Briefing

Auto-generated file at `.axon/briefing.md` (project) and `~/.axon/briefing.md` (global).

### Structure

```markdown
# User
- Name: Dhruv Jain
- Prefers TypeScript, pnpm, biome
- Hates docstrings on unchanged code, em-dashes
- Wants early returns over nested if-else

# Project: axon
- Memory engine CLI, TypeScript + Ink + commander
- Two-layer storage: project + global
- vitest for testing, biome for linting, tsup for builds

# Conventions
- Tab indentation, 100 char line width
- Never commit directly - tell user the commit message
- Run only relevant tests with it.only

# Recent Decisions
- Removed memory types - tags are enough
- Removed MCP server - CLI only
- Brain dump mode is default (axon with no args)

# Lessons
- Don't add error handling for impossible scenarios
- Check file exists before Write tool
- Use node: protocol for Node.js builtins
```

### Generation

Generated by Claude Code during consolidation. The skill:
1. Reads all memories from the archive
2. Groups by category (identity, convention, decision, lesson)
3. Compresses into structured markdown
4. Writes to briefing.md

Global briefing covers user identity + cross-project preferences.
Project briefing covers project-specific conventions + decisions + lessons.

## Tier 2: The Archive

### Memory File Format

```yaml
---
id: ax_abc123defg
tags: [auth, jwt, security]
category: convention
created: 2026-03-19T10:00:00Z
updated: 2026-03-19T10:00:00Z
accessed: 2026-03-19T10:00:00Z
accessCount: 3
summary: Auth uses JWT with refresh tokens in httpOnly cookies
---

Authentication is handled via JWT tokens.
- Access tokens: 15min expiry, stored in memory
- Refresh tokens: 7d expiry, httpOnly cookie
- Middleware at src/middleware/auth.ts validates on every request
```

### Categories (system-level, not user-facing)

| Category | What it captures | Briefing section |
|----------|-----------------|------------------|
| `identity` | User name, preferences, style | # User |
| `convention` | Coding patterns, project structure, tools | # Conventions |
| `decision` | Architectural choices, trade-offs, reasoning | # Recent Decisions |
| `lesson` | Corrections, mistakes to avoid, gotchas | # Lessons |

Default category when not specified: `convention`.

### File Layout

```

  search.json         # BM25 inverted index
  briefing.md         # Tier 1 generated briefing

    ax_abc123.md
    ax_def456.md

~/.axon/
  index.json
```

## BM25 Retrieval

### Inverted Index (search.json)

```json
{
  "version": 1,
  "docCount": 15,
  "avgDocLength": 42,
  "terms": {
    "auth": { "df": 3, "postings": { "ax_abc": 2, "ax_def": 1 } },
    "jwt": { "df": 2, "postings": { "ax_abc": 3 } },
    "token": { "df": 4, "postings": { "ax_abc": 2, "ax_ghi": 1, "ax_jkl": 1 } }
  }
}
```

### Scoring

```
finalScore = BM25(query, doc) * recencyBoost * accessBoost
```

- BM25 parameters: k1=1.5, b=0.75 (standard defaults)
- Recency boost: exponential decay, half-life 30 days
- Access boost: log(accessCount + 1) normalized

### Tokenization

Simple whitespace + punctuation split, lowercase, stop word removal. No stemming for v2 - keep it simple, add later if needed.

### Index Updates

Both `index.json` and `search.json` update atomically on every write. `axon reindex` rebuilds both from markdown files.

## Claude Code Skill

### Trigger

The skill is passive - it activates on EVERY conversation. It's not invoked with a slash command. It's always-on context injection + behavior modification.

### Skill Instructions (SKILL.md)

The skill instructs Claude to:

**1. On conversation start:**
- Read `~/.axon/briefing.md` and `.axon/briefing.md` (if they exist)
- Inject their contents as context
- Note: "You have access to axon memory via the axon CLI"

**2. During conversation - auto-capture:**
When Claude detects any of these signals, it runs `axon remember "..." --tags "x,y" --category <cat>` and shows the brain emoji:

| Signal | Category | Capture rule |
|--------|----------|-------------|
| User corrects Claude's output | `lesson` | "Never do X" / "Always do Y instead" |
| User states a preference | `identity` | "I prefer X" / "Always use X" |
| User or Claude makes an architectural decision | `decision` | "Using X because Y" |
| A pattern is used/referenced 3+ times | `convention` | "Standard pattern: X" |
| A bug root cause is identified | `lesson` | "Bug was caused by X" |
| User explicitly says "remember this" | varies | Capture whatever follows |

**3. When recalling deeper context:**
If the briefing doesn't have enough info for the current task, Claude runs `axon recall "query"` to search the archive.

**4. On conversation end:**
Claude runs consolidation:
1. `axon recall --recent 50` to get all recent memories
2. Review for duplicates, contradictions, outdated entries
3. Merge/update via `axon forget` + `axon remember`
4. Read ALL memories and regenerate briefing.md
5. Write updated briefing to `.axon/briefing.md` and/or `~/.axon/briefing.md`

**5. Visual indicator:**
Every time axon is called (remember, recall, forget, consolidate), Claude prefixes the action with the brain emoji in its output.

## CLI Commands (v2)

```bash
# Core
axon                          # brain dump mode (interactive, no args)
axon remember [text]          # store a memory (interactive wizard if no text)
axon recall [query]           # search with BM25 (interactive if no query)
axon forget [id]              # delete (interactive selector if no id)
axon status                   # counts, category breakdown, briefing freshness
axon briefing                 # show current Tier 1 briefing
axon reindex                  # rebuild index.json + search.json from files

# Setup
axon init                     # interactive: project/global/both

# Brain dump slash commands
/recall <query>               # search memories
/forget <id>                  # delete memory
/status                       # memory counts
/briefing                     # show briefing
/help                         # show commands
```

**Removed from v1:**
- `consolidate` command (now LLM-powered via skill)
- `prune` command (handled during consolidation)
- `importance` field (BM25 + access count handles ranking)
- `--metadata` flag on recall (metadata hidden by default, shown when explicitly needed)
- MCP server + all MCP code

**New:**
- `axon briefing` command
- `--category` flag on remember (used by skill, hidden from help)
- `search.json` BM25 index
- `briefing.md` auto-generated file

- Existing memories keep working - `category` defaults to `convention` if missing
- `importance` field in old files is ignored
- `search.json` generated on first `axon reindex` or first write
- `briefing.md` generated on first consolidation via skill
- No breaking changes to file format - additive only
